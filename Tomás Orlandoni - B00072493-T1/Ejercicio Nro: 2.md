# Ejercicio Nro: 2

## Enunciado
¿Qué es para vosotros el desarrollo de software?

- ¿Ejemplos de desarrollo de software en los que hayas participado? ¿Ejemplos que conozcas?
- ¿Cuál es el papel del desarrollador en el desarrollo de un proyecto software?
- ¿Hay otros papeles importantes en el desarrollo, aparte del desarrollador?
- ¿Cuándo se ha desarrollado un buen software?

## Resolución

Para mí, el desarrollo de software es el proceso completo de creación, diseño, despliegue y soporte de programas informáticos que sirven para resolver un problema específico o satisfacer una necesidad a través de la tecnología.

Realicé un software para la materia de Ingeniería de Software el año pasado, el programa consistía de venta y stock de productos de hardware.

El desarrollador no es solo alguien que tipea código frente a una pantalla, sino un solucionador de problemas integral que analiza lo que se necesita, diseña cómo construirlo, lo programa, busca y corrige errores, y trabaja en equipo para asegurar que el producto final realmente funcione y mejore continuamente.

Sí, están:
- el Product Owner, que representa al cliente y define qué priorizar;
- los diseñadores UX/UI, que arman la estructura y el aspecto visual para que el sistema sea fácil e intuitivo de usar;
- los analistas de QA, que prueban meticulosamente el software para cazar errores antes del lanzamiento;
- el Scrum Master, que destraba problemas diarios y ayuda a que el equipo trabaje sin fricciones;
- y los perfiles de DevOps, que gestionan la infraestructura técnica y automatizan los procesos para que el código llegue a los usuarios finales de manera rápida y segura.

Un software se considera "bueno" cuando logra un equilibrio entre utilidad, usabilidad y calidad técnica. Porque no se trata únicamente de que el programa no tenga errores, sino de que realmente resuelva un problema específico del usuario.
